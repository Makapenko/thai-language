# План исправления семантических ошибок в генераторе фраз

## Проблема

Генератор создаёт семантически некорректные фразы путём полного декартова произведения подлежащее × глагол × объект:

- ❌ `Вы (вежл.) не смотрели окно` — «смотреть окно» не имеет смысла
- ❌ `Вы (вежл.) открываете стул?` — «открывать стул» бессмыслица
- ❌ `Ты (неформ.) смотришь стену` — грамматически верно, но семантически странно

**Корень проблемы**: генератор комбинирует все слова со всеми без семантической валидации.

---

## Варианты решения

### Вариант 1. Ручная валидация глагол-существительных пар (рекомендуемый)

**Суть**: добавить таблицу совместимости глагол × существительное.

**Плюсы**:
- ✅ Полный контроль над семантикой
- ✅ Можно добавлять исключения и нюансы
- ✅ Не меняет архитектуру генератора

**Минусы**:
- ⚠️ Требует ручной работы при добавлении новых слов
- ⚠️ Таблица может расти (10 глаголов × 6 существительных = 60 записей)

**Реализация**:

```typescript
// src/data/vocabulary/verbObjectCompatibility.ts

// Карта совместимости: thai verb → array of compatible noun thai
export const verbObjectCompatibility: Record<string, string[]> = {
  'เปิด': ['ประตู', 'หน้าต่าง', 'หนังสือ', 'กล่อง'],      // открыть
  'ปิด': ['ประตู', 'หน้าต่าง', 'книгу', 'กล่อง'],         // закрыть
  'เห็น': ['คน', 'สุนัข', 'บ้าน', 'รถ', 'ประตู', 'หน้าต่าง'], // видеть
  'ดู': ['ทีวี', 'หนัง', 'คน', 'สุนัข', 'บ้าน'],           // смотреть
  'ซื้อ': ['บ้าน', 'รถ', 'หนังสือ', 'อาหาร', 'เสื้อ'],     // покупать
  'ทำ': ['งาน', 'อาหาร', 'หนังสือ', 'บ้าน'],               // делать
  'เปลี่ยน': ['เสื้อ', 'บ้าน', 'รถ', 'หนังสือ'],           // менять
  'สร้าง': ['บ้าน', 'รถ'],                                 // строить
  'หา': ['งาน', 'เงิน', 'อาหาร', 'คน'],                    // искать
  'ตัด': ['เสื้อ', 'ผม', 'กระดาษ'],                        // резать/стричь
};

// Функция проверки
export function isVerbObjectCompatible(verbThai: string, nounThai: string): boolean {
  const compatibleNouns = verbObjectCompatibility[verbThai];
  if (!compatibleNouns) return true; // если нет ограничений — разрешаем
  return compatibleNouns.includes(nounThai);
}
```

**Изменения в генераторе**:

```typescript
// src/data/phraseGenerator.ts

import { isVerbObjectCompatible } from './vocabulary/verbObjectCompatibility';

export function generatePhrasesWithObjects(...): Phrase[] {
  // ...
  for (const pattern of selectedPatterns) {
    for (const subject of subjects) {
      for (const verb of verbs) {
        for (const noun of nouns) {
          // НОВАЯ ПРОВЕРКА
          if (!isVerbObjectCompatible(verb.thai, noun.thai)) {
            continue; // пропускаем несовместимые пары
          }
          
          phrases.push({ ... });
        }
      }
    }
  }
}
```

### Вариант 2. Категории глаголов и существительных

**Суть**: пометить глаголы и существительные категориями, разрешать только совместимые.

**Категории глаголов**:
- `opening_closing` — открывать/закрывать
- `perception` — видеть/смотреть
- `creation` — делать/строить/создавать
- `acquisition` — покупать/получать
- `change` — менять/заменять

**Категории существительных**:
- `container` — дверь, окно, коробка
- `living_thing` — человек, собака
- `structure` — дом, здание
- `vehicle` — машина
- `abstract` — работа, деньги

**Правила совместимости**:
```typescript
const categoryCompatibility: Record<string, string[]> = {
  'opening_closing': ['container'],
  'perception': ['living_thing', 'structure', 'container'],
  'creation': ['structure', 'abstract'],
  'acquisition': ['structure', 'vehicle', 'container', 'abstract'],
  'change': ['structure', 'vehicle', 'container'],
};
```

**Плюсы**:
- ✅ Масштабируемо — добавляешь слова в категории, не правила
- ✅ Меньше ручной работы при росте словаря

**Минусы**:
- ⚠️ Менее точный контроль (нельзя учесть нюансы между конкретными словами)
- ⚠️ Сложнее реализовать

### Вариант 3. Ручной список допустимых фраз

**Суть**: не генерировать автоматически, а вручную составить список фраз.

**Плюсы**:
- ✅ Полный контроль над каждой фразой
- ✅ Можно добавить контекст и примеры

**Минусы**:
- ❌ Не масштабируется
- ❌ Теряется преимущество генератора
- ❌ Сложно поддерживать

**Не рекомендуется** для текущего проекта.
