## Проблема 1: Рефлексивные (бессмысленные) фразы

### Описание

Генератор создаёт предложения, где подлежащее и дополнение — одно и то же лицо. Это создаёт бессмысленные конструкции типа "Я знаю себя" (вместо "Я знаю тебя").

### Примеры

| Русский перевод | Тайский текст | Проблема |
|----------------|---------------|----------|
| Я (жен., форм.) знаю меня | ดิฉันรู้ดิฉัน | Подлежащее = Дополнение |
| Я (жен., форм.) вижу меня | ดิฉันเห็นดิฉัน | Подлежащее = Дополнение |
| Я помогаю меня | ดิฉันช่วยดิฉัน | Подлежащее = Дополнение |
| Ты (грубое) любишь тебя | มึงรักมึง | Подлежащее = Дополнение |

**Количество:** 624 фразы (все бессмысленные)

### Причина

Все местоимения из `objectPronouns` включают те же слова, что и `pronouns` (подлежащие):

```typescript
// Подлежащие (lesson2Pronouns)
{ thai: 'ดิฉัน', ... } // "я (жен., форм.)"

// Дополнения (objectPronouns)
{ thai: 'ดิฉัน', ... } // "меня" ← то же самое слово!
```

Генератор создаёт все возможные комбинации без фильтрации.

### Решение

**Вариант A — Фильтрация при генерации:**
Добавить проверку в `generatePhrasesWithPronounObjects`, чтобы пропускать комбинации где `subject.thai === objectPronoun.thai`.

**Плюсы:**
- Простая реализация
- Уменьшит на 624 фразы

**Минусы:**
- Не решает проблему стилистических конфликтов

**Вариант B — Разделить местоимения-дополнения:**
Создать отдельные массивы дополнений для каждого подлежащего, исключая рефлексивные комбинации.

**Рекомендация:** Использовать **Вариант A** (фильтрация при генерации) — проще и эффективнее.

## Проблема 2: Неправильный падеж для глагола "давать" (ให้)

### Описание

Глагол `ให้` (hâi — давать) в русском языке требует **дательного падежа** (кому? чему?), а не винительного (кого? что?).

В текущей реализации все дополнения используют `russianAccusative` (винительный падеж), что создаёт неправильные переводы.

### Примеры

| Неправильно | Правильно | Тайский текст |
|------------|-----------|---------------|
| Я даю **тебя** | Я даю **тебе** | ฉันให้คุณ |
| Ты даёшь **её** | Ты даёшь **ей** | เธอให้เขา |
| Он даёт **меня** | Он даёт **мне** | เขาให้ผม |
| Мы даём **их** | Мы даём **им** | เราให้พวกเขา |

**Количество:** ~600 фраз с глаголом `ให้`

### Причина

Все паттерны используют `o.russianAccusative` (винительный падеж):

```typescript
russianTemplate: (s, v, o) => capitalizeFirst(`${getSubjectLabel(s)} ${v.present[s.conjIndex]} ${o.russianAccusative}`)
//                                                                                    ^^^^^^^^^^^^^^^^
//                                                                      Всегда использует винительный падеж
```

Но разные глаголы требуют разные падежи:
- **Винительный** (кого?): видеть, любить, знать, слышать, встречать
- **Дательный** (кому?): давать, помогать, звонить, отвечать, спрашивать

### Решение

**Вариант A — Добавить dative форму в ObjectPronoun:**

```typescript
export interface ObjectPronoun {
  thai: string;
  transcription: string;
  russian: string;
  russianAccusative: string;  // винительный: меня, тебя, его
  russianDative: string;      // дательный: мне, тебе, ему
}
```

И паттерны будут выбирать нужный падеж в зависимости от глагола:

```typescript
// Определяем какие глаголы требуют дательный падеж
const dativeVerbs = ['ให้', 'ช่วย', 'โทร', 'ตอบ', 'ถาม'];

russianTemplate: (s, v, o) => {
  const objectForm = dativeVerbs.includes(v.thai) ? o.russianDative : o.russianAccusative;
  return capitalizeFirst(`${getSubjectLabel(s)} ${v.present[s.conjIndex]} ${objectForm}`);
}
```

**Плюсы:**
- Точный перевод для каждого глагола
- Масштабируемое решение

**Минусы:**
- Требует изменения интерфейса
- Нужно обновить все данные objectPronouns

**Вариант B — Исключить глаголы с дательным падежом:**

Убрать из списка глаголов: `ให้`, `ช่วย`, `โทร`, `ตอบ`, `ถาม`

**Плюсы:**
- Быстрое решение

**Минусы:**
- Потеряем 5 важных глаголов (~3000 фраз)
- Эти глаголы очень полезны для практики

**Вариант C — Создать отдельные паттерны:**

Создать `patternsVerbalObject` для глаголов с дательным падежом.

**Рекомендация:** Использовать **Вариант A** — добавить `russianDative` в интерфейс. Это наиболее правильное решение с точки зрения лингвистики.

## Проблема 3: Стилистические конфликты (регистр вежливости)

### Описание

В тайском языке разные местоимения имеют разный уровень вежливости. Генератор смешивает формальные, обычные и грубые местоимения в одном предложении, что создаёт стилистически некорректные фразы.

### Примеры

| Русский перевод | Тайский текст | Проблема |
|----------------|---------------|----------|
| Я (жен., форм.) знаю **тебя (грубое)** | ดิฉันรู้มึง | Формальное + грубое |
| **Ты (грубое)** помогаю **вам (вежл.)** | มึงช่วยท่าน | Грубое + вежливое |
| **Я (грубое)** вижу **вас (вежл.)** | กูเห็นท่าน | Грубое + вежливое |

**Количество:** Оценочно 1500-2000 фраз

### Причина

Все местоимения используются без учёта регистра:

```typescript
// Subject (грубый)
{ thai: 'กู', ... } // "я (грубое)"

// Object (вежливый)
{ thai: 'ท่าน', ... } // "вас (вежл.)"

// Результат: กูเห็นท่าน = "Я (грубое) вижу вас (вежл.)" ← бессмыслица!
```

В реальной речи тайцы никогда не смешивают:
- Если говорят **กู-มึง** (грубое) — оба местоимения грубые
- Если говорят **ดิฉัน-ท่าน** (формальное) — оба формальные
- Смешение = стилистическая ошибка

### Решение

**Вариант A — Добавить поле register и фильтровать:**

```typescript
export interface ObjectPronoun {
  thai: string;
  transcription: string;
  russian: string;
  russianAccusative: string;
  russianDative?: string;
  register: 'formal' | 'neutral' | 'informal' | 'rude';  // ← НОВОЕ
}
```

И при генерации проверять совместимость:

```typescript
const compatibleRegisters = {
  'formal': ['formal', 'neutral'],
  'neutral': ['formal', 'neutral', 'informal'],
  'informal': ['neutral', 'informal', 'rude'],
  'rude': ['informal', 'rude'],
};

// В генераторе:
if (!compatibleRegisters[subject.register].includes(objectPronoun.register)) {
  continue; // пропустить несовместимую комбинацию
}
```

**Плюсы:**
- Правильное стилистическое распределение
- Гибкая система совместимости

**Минусы:**
- Требует изменения интерфейса
- Нужно добавить register ко всем местоимениям

**Вариант B — Создать таблицы совместимости:**

Создать явный список разрешённых комбинаций subject-object.

**Плюсы:**
- Полный контроль над каждой комбинацией

**Минусы:**
- Много ручной работы
- Сложно поддерживать

**Вариант C — Исключить проблемные местоимения:**

Убрать грубые местоимения (กู, มึง) и очень формальные (ดิฉัน, ท่าน) из генерации, оставить только нейтральные.

**Плюсы:**
- Быстрое решение

**Минусы:**
- Потеряем много полезных фраз
- Пользователи не научатся различать регистры

**Рекомендация:** Использовать **Вариант A** — добавить поле `register`. Это позволит правильно фильтровать комбинации и обучать пользователей стилистике.

## Проблема 4: Глагол ต้องการ (хотеть/нуждаться) — двусмысленность

### Описание

Глагол `ต้องการ` (dtông-gaan) означает "хотеть" или "нуждаться". При использовании с местоимениями-людьми может создавать двусмысленные или неловкие фразы.

### Примеры

| Русский перевод | Тайский текст | Проблема |
|----------------|---------------|----------|
| Я хочу тебя | ฉันต้องการคุณ | Двусмысленный/романтический подтекст |
| Ты (грубое) хочешь его | มึงต้องการเขา | Грубое + романтический |
| Я (жен.) хочу вас (вежл.) | ดิฉันต้องการท่าน | Формальное + романтический |

**Количество:** ~600 фраз

### Причина

В английском "I want you" может означать:
1. "Я хочу, чтобы ты что-то сделал" (нейтральное)
2. "Я желаю тебя" (романтическое/сексуальное)

В тайском `ต้องการ` + человек чаще воспринимается как романтический интерес, а не просто "хотеть".

### Решение

**Вариант A — Убрать глагол ต้องการ:**

Исключить из списка глаголов для этого типа предложений.

**Плюсы:**
- Быстрое решение
- Исключает двусмысленность

**Минусы:**
- Потеряем полезный глагол

**Вариант B — Оставить только с формальными местоимениями:**

Использовать `ต้องการ` только с формальными/вежливыми комбинациями, где контекст понятен.

**Плюсы:**
- Сохраняем глагол для полезных случаев

**Минусы:**
- Сложная логика фильтрации

**Вариант C — Изменить перевод:**

Использовать "Я нуждаюсь в тебе" или "Мне нужен ты" вместо "Я хочу тебя".

**Плюсы:**
- Более нейтральный перевод

**Минусы:**
- Не совсем точный перевод

**Рекомендация:** Использовать **Вариант A** — убрать глагол `ต้องการ` из этого типа предложений. Его можно использовать в других контекстах (например, с объектами-вещами: "ฉันต้องการน้ำ" — "Я хочу воды").
